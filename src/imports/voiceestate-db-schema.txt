You are an expert full-stack developer. Help me build a multi-tenant SaaS platform called "VoiceEstate" — an AI-powered outbound voice calling platform for real estate businesses in Pakistan. The AI agent calls clients on behalf of real estate companies to pitch projects, capture interest, and book site visits. The voice agent is powered by VAPI.ai.

---

## TECH STACK

Frontend: Next.js 14 (App Router), TypeScript, Tailwind CSS, shadcn/ui
Backend: Node.js with Express (TypeScript)
Database: PostgreSQL with Prisma ORM
Cache: Redis (via Upstash)
File Storage: AWS S3 (CSV imports, recordings)
Voice Agent: VAPI.ai (outbound calls)
Auth: NextAuth.js with JWT
Payments: Not in scope yet
Deployment: Docker ready

## VAPI INTEGRATION

In /lib/vapi.ts:
- Initialize VAPI client using VAPI_API_KEY from env
- Function: createOutboundCall(clientPhone, assistantConfig) → calls VAPI /call/phone endpoint
- Function: buildAssistantConfig(campaign, client) → builds the full VAPI assistant object with:
  - model: gpt-4o
  - voice: ElevenLabs (voice_id from campaign)
  - system prompt (inject campaign variables: company name, project name, price, USP, FAQ, client first name, payment plan)
  - firstMessage: "Hi, am I speaking with {client.name}?"
  - language detection instruction in the prompt
  - endCallMessage on completion

In /api/campaigns/[id]/trigger:
- Validate campaign is active and within calling hours
- Fetch all CampaignClients where call_status is pending or retry_eligible
- Respect concurrency limit using a Redis queue (BullMQ)
- For each client in queue: call createOutboundCall, save vapi_call_id to CallLog
- If outside calling hours: return error "Outside allowed calling window"

In /api/vapi (POST webhook):
- Handle VAPI webhook events:
  - call-started → update CallLog status
  - call-ended → update duration, status, save transcript, recording_url
  - transcript-ready → run sentiment/outcome detection, update CampaignClient call_result
  - function-call (if using VAPI tools) → handle appointment booking trigger
- Auto-create Lead record if outcome is "interested"
- Auto-create Appointment if agent booked a visit during call

## AI PROMPT TEMPLATE (inject into VAPI assistant system prompt)

Use this as the base, with dynamic variables replaced per campaign:

"""
You are Zara, a professional real estate sales representative calling on behalf of {{company_name}}. You are calling about {{project_name}}.

Speak in the language the client uses. If they speak Urdu, respond in Urdu. If English, respond in English. If mixed, mix both naturally. Never ask which language they prefer.

Your tone is confident, warm, and persuasive. You have 3-4 minutes. Stay focused.

TASK 1 - GREETING:
Greet the client by first name. Confirm identity. Introduce yourself and the project in one compelling line.

TASK 2 - PROJECT PITCH:
Cover: what it is, location, developer, key USP, payment plan. End with a qualifying question.
Project details: {{project_description}}. Starting price: {{starting_price}}. Payment plan: {{payment_plan}}. Location: {{location}}. USP: {{usp}}.

TASK 3 - QUALIFY:
Ask: buying to live or invest? Budget? Actively looking or exploring? One question at a time. Internally classify as hot, warm, or cold.

TASK 4 - FAQ:
Answer from this list: {{faq_list}}. If unknown, say the team will follow up. Always redirect back to closing after answering.

TASK 5 - OBJECTIONS:
Price too high → highlight payment plan flexibility.
Location concerns → highlight future development and accessibility.
Need to think → suggest site visit to make decision easier.
Not interested → ask if timing or project issue, log reason, close warmly.

TASK 6 - CLOSE:
Try to book a site visit first. If not, book a callback. If not, offer WhatsApp details.
Say: "We have slots this Saturday or next Tuesday - which works for you?"
If they agree: confirm date, time, their number.

TASK 7 - CLOSING:
Summarize next step in one sentence. Thank them. End warm.

TASK 8 - FALLBACK:
If confused after 2 attempts: "Let me have one of our team members reach out to assist you better."
"""

## RULES FOR DEVELOPMENT

- Every API route must validate company_id from session (no cross-tenant data leaks)
- All forms use react-hook-form + zod validation
- Use server actions where possible in Next.js App Router
- All tables are paginated (default 25 per page)
- Loading states on all async actions
- Toast notifications on success/error
- Mobile responsive (Tailwind)
- All VAPI calls are logged immediately, webhook updates the record async
- Redis queue (BullMQ) handles campaign call dispatch with concurrency control
- Calling hours enforced server-side, not just UI
- CSV import validates phone format before inserting
- Duplicate phone detection on import and manual entry
